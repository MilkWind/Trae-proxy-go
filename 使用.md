# 使用文档

## 第一步：安装 OpenSSL

证书生成依赖 OpenSSL，请确保系统已安装并加入 PATH。

**Windows**

前往 [https://slproweb.com/products/Win32OpenSSL.html](https://slproweb.com/products/Win32OpenSSL.html) 下载安装包（推荐 Win64 OpenSSL v3.x Light 版本），安装时勾选「Copy to Windows system directory」。

安装完成后打开命令提示符验证：

```bash
openssl version
```

输出版本号则表示安装成功。

**macOS**

```bash
brew install openssl
```

**Linux**

```bash
sudo apt install openssl   # Debian / Ubuntu
sudo yum install openssl   # CentOS / RHEL
```

---

## 第二步：启动服务，签发证书

### 启动代理服务

```bash
运行 cmd/proxy/main.go
```

代理服务启动后，同时会在 `8080` 端口开启 Web 管理界面。

### 打开管理页面

浏览器访问：

```
http://localhost:8080
```

### 签发证书

1. 在管理页面的「证书与多域名管理」区域，确认域名列表中包含需要代理的域名（如 `api.openai.com`、`api.anthropic.com`）
2. 点击「生成证书」按钮，程序会调用 OpenSSL 在 `ca/` 目录下生成：
   - `ca/ca.crt` — 根 CA 证书
   - `ca/<domain>.crt` / `ca/<domain>.key` — 域名证书
3. 将 `ca/ca.crt` 安装到系统的受信任根证书存储中：
4. 安装好后重新运行

**Windows**：（一般不需要） 双击 `ca.crt` → 安装证书 → 本地计算机 → 将所有证书放入「受信任的根证书颁发机构」

**macOS**：
```bash
sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain ca/ca.crt
```

**Linux**：
```bash
sudo cp ca/ca.crt /usr/local/share/ca-certificates/trae-proxy.crt
sudo update-ca-certificates
```

## 第三步：写入 hosts 文件

在管理页面点击「写入 Hosts」按钮，程序会将代理域名解析到本机：

```
127.0.0.1  api.openai.com
127.0.0.1  api.anthropic.com
```

> 写入 hosts 文件需要管理员 / root 权限，如提示权限不足，请以管理员身份重新运行程序。

验证是否生效：

```bash
ping api.openai.com
```

返回 `127.0.0.1` 则表示成功。

---

## 第四步：添加模型

在管理页面的「API 配置」区域点击「添加」，填写以下字段：

| 字段 | 说明 | 示例 |
|---|---|---|
| 名称 | 配置的显示名称 | `deepseek-r1` |
| 格式 | 后端 API 类型 | `openai` 或 `anthropic` |
| 后端地址 | 目标 API 的根地址 | `https://api.deepseek.com` |
| 自定义模型 ID | 客户端发送的模型名称 | `deepseek-reasoner` |
| 目标模型 ID | 实际转发给后端的模型名称 | `deepseek-reasoner` |
| 流式模式 | 强制覆盖流式设置 | 留空跟随请求，填 `true`/`false` 强制覆盖 |
| 激活 | 是否启用此配置 | 勾选 |
| custom_api_key| 填写就覆盖trae里面的api key| sk-134|

填写完成后点击「保存配置」。

配置保存后，客户端即可通过原有的 OpenAI / Anthropic API 地址正常调用，请求会被透明代理到你配置的后端。
